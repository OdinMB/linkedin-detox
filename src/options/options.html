Privacy Policy

Last updated: March 27, 2026

LinkedIn Detox is designed with a simple principle: your data never leaves your device.

What the extension accesses

The extension reads the text content of posts in your LinkedIn feed. This text is analyzed locally on your device to determine whether it appears to be AI-generated. No post content, browsing activity, or personal information is ever transmitted to any external server.

Local processing

All detection happens entirely within your browser. The heuristic scorers run synchronous pattern matching in the content script. The optional semantic scorer runs a quantized MiniLM language model in a local offscreen document — the model is bundled with the extension and performs inference on-device. No text is sent to any API or cloud service.

Data storage

Your settings (enabled state, sensitivity threshold, display mode, and custom detection patterns) are stored using chrome.storage.sync, which syncs preferences across your signed-in Chrome browsers via your Google account. Session statistics (posts scanned, posts blocked) are stored in chrome.storage.local and do not sync. No data is stored anywhere else.

No collection, no tracking, no network requests

LinkedIn Detox does not collect, store, or transmit any personal data. There are no analytics, no telemetry, no cookies, and no third-party services. The extension does not track which posts you view, which posts are flagged, or any other browsing behavior. The extension makes zero network requests — all code and model files are bundled locally.

Permissions explained

  • Host permissions (linkedin.com) — Read feed post text and modify the page to hide or replace flagged posts.
  • Storage — Persist your settings and session stats locally.
  • Offscreen — Run the local ML model for semantic detection in a background document (the service worker lacks the required WebAssembly support).

Open source

The full source code is available at github.com/OdinMB/linkedin-detox. You can verify every claim on this page by reading the code.
