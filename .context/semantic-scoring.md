# Semantic Scoring Architecture

## Overview

Optional fourth scorer that uses a local embedding model to detect AI-slop phrase *types* regardless of exact wording. Disabled by default ‚Äî users opt in via the popup toggle.

## Components

### semantic-scorer.js
Pure scoring logic (no browser APIs). Exports:
- `cosineSimilarity(a, b)` ‚Äî vector math
- `scoreFromSimilarity(similarity)` ‚Äî converts cosine similarity to 0-100 score (>=0.75 ‚Üí 80+, >=0.60 ‚Üí 50+, <0.60 ‚Üí 0)
- `computeSemanticScore(sentenceEmbeddings, phraseBank)` ‚Äî returns `{ score, matches }` matching the heuristic scorer interface

### background.js + offscreen.js
The embedding model needs full browser APIs (WebAssembly, Workers, Atomics) that MV3 service workers lack. Architecture:
- `background.js` ‚Äî thin relay service worker, creates offscreen document on first embed request
- `offscreen.js` ‚Äî loads `@xenova/transformers` with `Xenova/all-MiniLM-L6-v2` (quantized int8, bundled in `src/models/`), lazy-initializes on first message. Remote model fetching is disabled (`env.allowRemoteModels = false`).
- `semantic-bridge.js` ‚Äî content script helper that sends embed requests via `chrome.runtime.sendMessage`

### phrase-embeddings.json
Precomputed 384-dimensional embeddings for ~50 canonical AI-slop phrase types. Generated by `scripts/build-embeddings.js`. Must be regenerated when phrases change.

## Data Flow

1. `scanner.js` calls `analyzePostAsync(text, config)` for each post
2. `analyzePostAsync` (in `detector.js`) runs heuristic scorers synchronously (Pass 1) ‚Äî if the post is blocked, it returns immediately without invoking the semantic model
3. For uncaught posts when `semanticEnabled` and `config.getSemanticScore` is provided: `analyzePostAsync` calls `getSemanticScore(text)` ‚Üí `semantic-bridge.js` ‚Üí `chrome.runtime.sendMessage` ‚Üí `background.js` ‚Üí offscreen document ‚Üí model embeds sentences ‚Üí `computeSemanticScore` against phrase bank ‚Üí blocks if threshold met (Pass 2)
4. `content.js` injects `getSemanticScore` into the config when `semanticEnabled` is true

## Bundled Assets

All semantic scoring assets are checked into git ‚Äî no setup required:

- `src/lib/transformers.min.js` ‚Äî vendored `@xenova/transformers` CJS bundle (~877KB)
- `src/models/Xenova/all-MiniLM-L6-v2/` ‚Äî quantized MiniLM model files (~23MB)
- `src/phrase-embeddings.json` ‚Äî precomputed 384-dim embeddings for ~50 canonical phrase types (~530KB)

Re-run `node scripts/build-embeddings.js` whenever canonical phrases change.

## Key Decisions

- **Disabled by default** because it adds latency (model loads from bundled files on first activation)
- **Two-pass async** so heuristic-caught posts render instantly with no delay ‚Äî the expensive model is only invoked for posts the cheap heuristics missed
- **Offscreen document** hosts the model because MV3 service workers lack WebAssembly/Workers/Atomics support. Content scripts can't create extension-origin Workers either.
- **Vendored transformers.js** in `src/lib/transformers.min.js` ‚Äî MV3 CSP blocks CDN imports, so the library must be local. Manifest includes `'wasm-unsafe-eval'` for the ONNX runtime.
- **No bundler** ‚Äî matches project convention of no build step
